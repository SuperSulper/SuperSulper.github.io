# 슈퍼설퍼.github.io

React와 Supabase를 사용한 기본 페이지입니다.

```bash
npm install
npm run dev
```

`.env.local`에 Supabase 값을 넣어 실행합니다.

```bash
VITE_SUPABASE_URL=https://your-project-ref.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
```

Supabase SQL Editor에서 `entries` 테이블과 데모용 RLS 정책을 만들 수 있습니다.

```sql
create table if not exists public.entries (
  id bigint generated by default as identity primary key,
  label text not null,
  detail text not null,
  created_at timestamptz not null default now()
);

alter table public.entries enable row level security;

drop policy if exists "Anyone can read entries" on public.entries;
create policy "Anyone can read entries"
  on public.entries
  for select
  to anon
  using (true);

drop policy if exists "Anyone can create entries" on public.entries;
create policy "Anyone can create entries"
  on public.entries
  for insert
  to anon
  with check (true);
```

## GitHub Pages 배포

GitHub 저장소에서 `Settings` → `Pages` → `Build and deployment` → `Source`를 `GitHub Actions`로 설정합니다.

`Settings` → `Secrets and variables` → `Actions`에서 아래 값을 추가합니다.

- Repository variable: `VITE_SUPABASE_URL`
- Repository secret 또는 variable: `VITE_SUPABASE_ANON_KEY`

`main` 브랜치에 push하면 GitHub Actions가 `npm ci`, `npm run build`를 실행한 뒤 `dist`를 GitHub Pages에 배포합니다.
